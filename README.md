# Semantic Log Search with Spark, PostgreSQL, and pgvector

This project builds a simple Big Data pipeline for semantic search over web server logs.

It uses:

- Apache Spark for log ingestion and preprocessing
- Sentence-Transformers with `all-MiniLM-L6-v2` for embeddings
- PostgreSQL with `pgvector` for vector storage and similarity search
- A small web interface for demonstration

## Project Structure

- `main.py` - cleans logs, generates embeddings, inserts them into PostgreSQL, and creates the vector index
- `search_logs.py` - command-line semantic search tool
- `web_app.py` - browser-based interface for semantic search and basic analysis
- `config.py` - loads values from `.env`
- `.env.example` - example configuration file
- `access.log/` - raw log dataset folder
- `logs_clean/` - cleaned parquet output generated by Spark

## Configuration

Create a `.env` file from the example:

```powershell
copy .env.example .env
```

Then edit `.env` with your PostgreSQL credentials:

```env
PG_HOST=localhost
PG_PORT=5432
PG_DBNAME=logs_db
PG_USER=postgres
PG_PASSWORD=your_password
MODEL_NAME_OR_PATH=all-MiniLM-L6-v2
WEB_PORT=8000
RAW_LOG_PATH=access.log/access.log
CLEAN_LOG_PATH=logs_clean
SAMPLE_FRACTION=0.05
RUN_SPARK_CLEANING=1
EMBEDDING_BATCH_SIZE=128
```

## PostgreSQL Setup

Create the database if it does not already exist:

```powershell
createdb -h localhost -p 5432 -U postgres logs_db
```

Inside `psql`, enable `pgvector`:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

The project uses `all-MiniLM-L6-v2`, which produces 384-dimensional embeddings, so the table uses:

```sql
embedding VECTOR(384)
```

## Run the Pipeline

From the project folder:

```powershell
cd C:\Users\admin\Desktop\tp-bda
venv\Scripts\python.exe main.py
```

`main.py` will:

- parse Apache logs with Spark
- normalize URLs and create semantic messages
- save the cleaned dataset to `logs_clean`
- generate embeddings with `all-MiniLM-L6-v2`
- create the `sematic_logs` table with analytical fields
- insert rows if the table is empty
- create an IVFFlat vector index
- create analytical indexes on status, timestamp, and message

The PostgreSQL table stores:

- `ip`
- `timestamp`
- `method`
- `url`
- `clean_url`
- `status`
- `size`
- `referrer`
- `user_agent`
- `message`
- `embedding`

If you already created an older table with only `timestamp`, `message`, and `embedding`, drop it before running the updated pipeline:

```sql
DROP TABLE IF EXISTS logs;
DROP TABLE IF EXISTS sematic_logs;
```

## Command-Line Search

```powershell
venv\Scripts\python.exe search_logs.py --query "GET image product" --limit 10
```

Use keyword-like natural language queries, for example:

```powershell
venv\Scripts\python.exe search_logs.py --query "similar product image requests"
```

## Web Interface

Start the web app:

```powershell
venv\Scripts\python.exe web_app.py
```

Open:

```text
http://127.0.0.1:8000
```

The web interface includes:

- semantic search
- keyword search
- recurrent HTTP error detection
- temporal evolution of HTTP errors
- database statistics

These views cover the required practical cases:

- find logs semantically similar to a critical error
- identify frequent error groups
- analyze temporal evolution of similar/error logs
- compare semantic search with keyword search
